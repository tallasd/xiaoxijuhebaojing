# 第一阶段实际落地方案

根据当前运营情况，第一阶段不先上公网服务器，先把中台部署在本地电脑。你的本地电脑同时承担三件事：

1. 运行客服消息中台。
2. 运行网易邮箱大师，用来人工查看所有收件箱。
3. 运行邮件采集器和云端浏览器里的网页插件，把消息汇总到中台。

## 你的真实场景

现在消息来源分成两类。

第一类是会发邮件的平台，例如 eBay、日本 Game Club 这类。它们有消息时会发邮件到不同邮箱，网易邮箱大师能统一收到。系统这边不读取网易邮箱大师界面，而是用每个邮箱的 IMAP 信息读取邮件，再把平台消息写入中台。

第二类是不会发邮件、只在网页里提示的平台，例如 Shopee 台湾、Shopee 马来西亚，以及 Discord 聊天环境。这类必须靠浏览器插件巡检网页红点、未读数字、弹窗和聊天内容。

## 第一阶段的结论

邮件只能解决“会发邮件的平台”的汇总问题，不能解决所有平台。

所以第一阶段采用混合方案：

- eBay：先用邮箱 IMAP 接入。
- 日本 Game Club：先用邮箱 IMAP 接入，同时建议安装插件做网页兜底。
- Shopee 台湾：必须安装浏览器插件。
- Shopee 马来西亚：必须安装浏览器插件。
- Discord：先按网页聊天插件监控处理。
- 网易邮箱大师：继续保留给人工查看，但中台程序不依赖它的界面。

## 数据怎么汇总

### 会发邮件的平台

```text
平台消息
  -> 平台发送提醒邮件
  -> 邮箱账号
  -> 中台电脑 IMAP 采集器
  -> 客服中台
```

这条链路适合 eBay、日本 Game Club。

### 不发邮件的平台

```text
平台网页
  -> 浏览器插件每 30 秒巡检
  -> 本地中台 API
  -> 客服中台
```

这条链路适合 Shopee 台湾、Shopee 马来西亚、Discord。

远程电脑、海外主机邮件中转先不作为当前重点。如果以后某些平台必须放在远程电脑、海外主机上运行，而不是和中台在同一台电脑，再改成：

```text
远程浏览器插件
  -> 远程电脑本地邮件中转
  -> SMTP 发到中台邮箱
  -> 中台电脑 IMAP 读取
  -> 客服中台
```

## 插件装在哪里

插件装在“实际打开平台网页的浏览器环境”里。

也就是说：

- Shopee 台湾 4 个店铺，各自所在的云端浏览器环境都要装。
- Shopee 马来西亚 1 个店铺，所在云端浏览器环境要装。
- Discord 两个聊天环境，各自所在环境要装。
- Game Club 日本 3 个店铺，如果网页也能看到未读和聊天，建议也装。

插件配置里要填：

```text
中台/本地中转地址：http://127.0.0.1:4100
巡检间隔：30秒
平台：比如 Shopee TW / Shopee MY / Game Club JP / Discord
店铺名：对应具体店铺
```

如果插件所在环境不能直接访问本地中台，就改填：

```text
http://127.0.0.1:4222
```

这表示先发给本机邮件中转，再通过邮箱回传到中台。

## 现在要优先做什么

1. 填 `docs/邮箱清单.csv`，先把 eBay 和 Game Club 的邮箱接入。
2. 填 `docs/运营环境清单.csv`，把 Shopee 台湾 4 个店铺、Shopee 马来西亚 1 个店铺、Discord 2 个环境补上具体名称。
3. 在 Shopee 和 Discord 所在云端浏览器里安装插件。
4. 每个插件配置平台名、店铺名、巡检间隔 30 秒。
5. 打开中台，看是否能出现“新消息提醒”和“质检告警”。

当前插件默认直连本机中台：

```text
http://127.0.0.1:4100
```

## 判断是否成功

成功以后，中台应该能看到：

- 哪个平台有新消息。
- 哪个店铺有新消息。
- 消息来自邮箱还是网页插件。
- 哪个客服有没有及时回复。
- 是否出现敷衍回复。
- 是否出现违规词。
- 哪个环境/电脑上报了消息。

第一阶段的目标不是自动回复，而是做到“发现、记录、提醒、告警”。只要能稳定发现消息，就已经解决你现在最痛的漏消息问题。
