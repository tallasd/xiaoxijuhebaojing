# 网页插件与网易邮箱大师接入方案

## 核心思路

系统分两条线发现消息：

1. 网页平台：在每个接待网页所在浏览器安装采集插件。
2. 邮件提醒平台：继续用网易邮箱大师统一收件，同时让中台通过 IMAP 或桌面通知接入邮件提醒。

中台不接管平台登录、不绕过验证码、不自动发消息。它只做发现、提醒、派单、质检和追踪。

## 网页插件负责什么

插件安装在 Shopee、G2G、GameTrade、GameClub、rmt.club、Etsy 后台、eBay 后台等网页环境里。

第一阶段监控：

- 未读数字、红点、浏览器标题变化。
- 消息列表新增。
- 客户聊天内容。
- 客服回复内容。
- 当前页面 URL，作为回原平台处理入口。

巡检节奏：

- 插件默认每 30 秒完整巡检一次。
- 页面 DOM 发生变化、浏览器标签重新聚焦时，会立即触发一次轻量扫描。
- 同一条消息、同一个客服回复、同一个质检告警会做去重，不会因为 30 秒巡检而一直重复报警。

质检判断：

- 客服是否回复过慢。
- 客服是否只回复 `ok`、`好的`、`稍等` 这类敷衍内容。
- 客服是否提到站外联系、私下付款、银行卡、WhatsApp、微信等敏感词。
- 客户消息是否长期没有检测到客服回复。

## 网易邮箱大师如何接入

你现在已经把不同平台、不同邮箱统一接到网易邮箱大师里，这是很好的“人工统一入口”。中台接入有两种方式。

### 方式 A：中台直接读各邮箱 IMAP

适合稳定长期使用。

每个邮箱在中台配置：

- IMAP 地址。
- 邮箱账号。
- 授权码。
- 对应平台。
- 对应店铺。
- 邮件标题/发件人/正文关键词规则。

优点：不依赖网易邮箱大师客户端是否打开，后台服务器可持续采集。

### 方式 B：监听网易邮箱大师桌面通知

适合第一阶段快速落地。

网易邮箱大师收到新邮件后，Windows 弹通知。桌面通知采集器捕获通知标题、正文、邮箱名、时间，然后上报中台。

优点：配置快，能先证明业务链路。

限制：通知内容不一定完整，后续仍建议切换到 IMAP。

## 推荐落地顺序

1. 先在 1-2 个网页平台安装浏览器插件，例如 Shopee 和 G2G。
2. 再接网易邮箱大师桌面通知，覆盖所有“有邮件提醒但网页提示不明显”的平台。
3. 对核心邮箱逐步改成 IMAP 直接采集。
4. 给每个平台配置更准确的聊天 DOM 选择器，减少误判。
5. 把质检规则扩展成后台可编辑：违规词、敷衍话术、回复 SLA、主管提醒规则。

## 现有代码位置

- 浏览器插件：`collectors/browser-extension`
- 邮件采集器：`collectors/email-imap`
- Windows 通知样例：`collectors/desktop-windows`
- 中台质检接口：`apps/api/src/quality.ts`
